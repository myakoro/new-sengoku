## 1. 概要

### 1-1. 設計方針

**Version 0.1の画面設計の核心：**
- **シンプル第一**：複雑なUIは避け、必要最小限の情報のみ表示
- **情報の優先順位**：プレイヤーが今知るべき情報を最優先で表示
- **選択の明確化**：何を選べるか、何が起こるかを明確に
- **フィードバックの即時性**：行動の結果をすぐに表示
- **緊張感の演出**：数値の変化、ライバルとの比較を視覚化

### 1-2. デザインコンセプト

```
【テーマ】
戦国時代の緊張感 × 現代的な見やすさ

【カラースキーム】
- ベース：ダークグレー（#2C2C2C）
- アクセント：金色（#D4AF37）
- 警告：赤（#C62828）
- 成功：緑（#2E7D32）
- テキスト：白（#FFFFFF）、薄灰色（#CCCCCC）

【フォント】
- 日本語：游ゴシック、メイリオ
- 数値：等幅フォント（Consolas、Monaco）
```

### 1-3. 技術スタック

```
フロントエンド：React + TypeScript
スタイリング：Tailwind CSS
状態管理：Zustand
ルーティング：React Router
アニメーション：Framer Motion
```

#### 要素

| セクション | 要素 | 説明 |
|-----------|------|------|

| | 経済 | 米、金、借金 |

| | 功績 | ライバルの功績 |

| ログ | メッセージ | ゲーム内の出来事を時系列表示 |

#### インタラクション

- **週を開始する**: 主命選択画面へ遷移
- **開始ターン固定の下知がある場合**: 該当ターンは主命選択へ進まず、下知の主命が自動開始される。
- **プレイヤー情報クリック**: ステータス詳細画面へ（オプション）
- **ライバル情報クリック**: ライバル詳細画面へ（オプション）

#### 盗賊情報（巡察）

- 巡察で盗賊の痕跡を発見した場合、メイン画面に「盗賊情報（巡察）」パネルを表示する。
- パネルには盗賊ランク、発見週（カレンダー週）を表示する。
- 各カードに対して以下の操作を提供する。
  - **討伐する（戦利品2倍）**: カード由来の盗賊討伐ミッションを開始する。
  - **報告する（功績+○）**: 上司へ上申し、討伐功績の40%を得てカードを消滅させる。
- 各カードに対して「若党委任（主人公ターン消費なし）」を提供する。
  - 担当若党を選択できる。
  - 委任できる内容：偵察／偽情報／買収／雇用。
  - 同一ターンの委任はカード全体で1回まで（既に委任済みのターンはグレーアウト）。
  - 偵察済みの盗賊に対する再偵察は無効（グレーアウト）。
  - 買収は「偵察済み」かつ「盗賊10人以上」で解禁。
- 経過（2か月でランク上昇、4か月で消滅）などのイベントは行動ログで通知される。

#### フロー

```
1. 収入表示（扶持米）
   ↓
2. 支出表示（若党、生活費、馬維持費）
   ↓
3. 余剰米の使い道選択
   - 全て売却（金に換金）
   - 半分売却
   - 売却しない（備蓄）
   ↓
4. 借金返済選択（借金がある場合のみ）
   - 全額返済
   - 半分返済
   - 少額返済（1貫）
   - 返済しない
   ↓
5. 状況表示（米、金、借金の最終状態）
   ↓
6. 出世判定（プレイヤー、ライバル）
   ↓
7. 評定（上司からのコメント、ライバルとの比較）
   ↓
8. メイン画面へ戻る
```

#### 盗賊討伐中の月次処理

盗賊討伐ミッション中（任務ターン数が残っている状態）に月次（4週ごと）が来た場合：

```
v0.1では、ミッション進行中に月次画面へ強制割り込みはしない。

1. 盗賊討伐画面で準備行動を選択 / 戦闘を実行
   ↓
2. 任務終了（結果を閉じる）または中止
   ↓
3. 【月次処理が発生】（月次境界のときのみ）
   ↓
4. メイン画面へ戻る
```

**注意**: v0.1では盗賊討伐中に月次画面へ割り込まないが、月次境界に到達していれば任務終了/中止で月次処理へ遷移する。

※月次報告（SC-06）は経済処理（扶持米・支出・借金など）を扱い、下知の裁定/発行は小頭評定（SC-05A）で行う。

## 8. 週数表示の実装

### 8-1. 表示形式

**採用形式**: 「1575年3月2週目」

```javascript
const GAME_START_YEAR = 1575

function formatWeek(week) {
  // 年の計算
  const yearsPassed = Math.floor((week - 1) / 52)
  const year = GAME_START_YEAR + yearsPassed
  
  // 月の計算（1週=1週間、4週=1ヶ月）
  const weekInYear = ((week - 1) % 52) + 1
  const month = Math.floor((weekInYear - 1) / 4) + 1
  
  // 月内の週（1〜4週目）
  const weekInMonth = ((weekInYear - 1) % 4) + 1
  
  return `${year}年${month}月${weekInMonth}週目`
}

// 例：
// week = 1  → "1575年1月1週目"
// week = 4  → "1575年1月4週目"
// week = 5  → "1575年2月1週目"
// week = 52 → "1575年12月4週目"
// week = 53 → "1576年1月1週目"
// week = 89 → "1576年9月1週目"
```

### 8-2. 表示理由

1. **戦国時代の臨場感**
   - 実際の年号を使うことで時代感が増す
   - 1575年は長篠の戦いの年（戦国時代の象徴的な年）

2. **月と週の両方が明確**
   - 季節感が分かる（1月=冬、3月=春など）
   - 週単位の進行も把握しやすい

3. **固定ゴールを示さない**
   - 年数表示なので終わりが見えない
   - 小頭到達でエンディング

### 8-3. 月と週の対応表

| 週（年内） | 月 | 週（月内） |
|-----------|----|-----------|
| 1〜4週 | 1月 | 1〜4週目 |
| 5〜8週 | 2月 | 1〜4週目 |
| 9〜12週 | 3月 | 1〜4週目 |
| ... | ... | ... |
| 49〜52週 | 12月 | 1〜4週目 |

### 8-4. 開始日時

```
ゲーム開始: 1575年1月1週目
```

**1575年を選んだ理由**:
- 長篠の戦いの年（戦国時代の象徴的な年）
- 実際の年号を使うことで時代感が増す
- 季節感が分かる（1月=冬、3月=春など）

### 12-3. 借金上限の定数

```javascript
// 借金上限（GDD準拠）
const DEBT_LIMIT = {
  徒士: 10,
  馬上衆: 50,
  小頭: 100
  // 組頭以降：無制限（Version 0.5以降）
}

// 借金可能かチェック
function canBorrow(player, amount) {
  const limit = DEBT_LIMIT[player.rank]
  return player.debt + amount <= limit
}

// 借金時のUI表示
function getDebtDisplay(player) {
  const limit = DEBT_LIMIT[player.rank]
  return `${player.debt.toFixed(1)}貫 / 上限: ${limit}貫`
}
```

### 12-5. 盗賊討伐中の月次処理

```javascript
function processWeek(player, mission) {
  // 週を進める
  player.week++
  
  // 4週ごとに月次処理
  if (player.week % 4 === 0) {
    // 盗賊討伐中でも月次処理を実行
    processMonthly(player)
    
    // 月次処理後、盗賊討伐画面に戻る
    if (mission && mission.type === "bandit_subjugation") {
      return { screen: "bandit_mission", mission }
    }
  }
  
  return { screen: "main" }
}
```
