# 戦国立身出世SLG プロジェクト経緯サマリー

**作成日：2025年12月5日**

---

## 1. プロジェクト概要

### ゲームコンセプト
戦国時代の一武士として、徒士から侍大将を目指す立身出世シミュレーションゲーム。

### 核心体験
**「立身出世の加速度を体感する」**
- 徒士期：何もない、リスクを取るしかない緊張感
- 馬上衆期：家臣団の萌芽、まだ経済的に厳しい
- 小頭期：ブレイクポイント、権限と資産が急増
- 以降：加速度的に権限・資産・影響力が増大

### 開発期間
2〜3年の長期プロジェクト

---

## 2. 失敗の教訓（v0.9までの開発）

### 何が起きたか
- v0.9まで実装完了（日常パート、家臣団、戦闘、経済、婚姻、引き抜き）
- UIも画面遷移も存在する
- **しかし「ゲームになっていない」**

### 問題の本質
```
パーツは全部ある：
✓ 役職システム
✓ 戦闘システム
✓ 経済システム
✓ 家臣団システム

でも欠けているもの：
✗ ゲームループ（短期・中期・長期の目標）
✗ 世界シミュレーション（プレイヤー以外が動く）
✗ 緊張感（リスクとプレッシャー）
✗ プレイヤー体験の設計

結果：
→ 画面があるけどゲームじゃない
→ 世界が動いてない
→ ただの作業
```

### 根本原因
**「システムのパーツカタログ」を作って、「ゲーム」を作らなかった**

- 要件定義書 = システムの羅列
- 開発ロードマップ = 機能の積み上げ順序
- 別紙A〜M = 詳細な世界設定とルール

**「プレイヤーが何をするゲームか」が一度も設計されなかった。**

## 3. 新しい開発方針

### 段階的開発（各Versionで完結）

#### Version 0.1（3ヶ月）
- 目標：徒士→馬上衆→小頭（ゲーム内3年間）
- プレイ時間：2〜3時間
- システム：出世システムのみ（最小限）
- ゴール：**「出世の喜び」が体感できること**

#### Version 0.5（+6ヶ月）
- 目標：小頭→組頭（ゲーム内5年間）
- プレイ時間：8〜10時間
- 追加：戦闘システム、家臣団、知行地
- ゴール：**「家臣を育て、戦で功を立てる」楽しさ**

#### Version 1.0（+12ヶ月）
- 目標：組頭→足軽大将→部将（ゲーム内10年間）
- プレイ時間：20〜30時間
- 追加：経済拡大、婚姻、引き抜き、世代交代
- ゴール：**「家を大きくし、戦国を生き抜く」体験**

#### Version 2.0（+?）
- 目標：侍大将→天下統一（ゲーム内30年間）
- プレイ時間：50時間以上
- 追加：戦略マップ、大名プレイ、外交
- ゴール：**別紙A〜Mの完全実装**

### 重要原則
1. **各Versionで「完結」させる** - 途中で止めても遊べる状態
2. **最小ループから段階的に** - システムを積む前にゲームを作る
3. **テストプレイ必須** - 面白くなければ方向転換
4. **別紙に縛られない** - 別紙は「辞書」として参照、絶対ではない

## 4. Version 0.1の詳細方針

### スコープ（何を作るか）

#### 実装するもの
- ✓ ターン制（1週間=1ターン、年間52ターン）
- ✓ 公務選択（3種類）
  - 訓練（功績+5、安全）
  - 盗賊討伐（功績+30、危険）
  - 巡察（功績+3、超安全）
- ✓ 功績累計システム
- ✓ 出世判定
  - 徒士→馬上衆（功績250）
  - 馬上衆→小頭（功績500）
- ✓ AIライバル1名（毎週行動、出世競争）
- ✓ 月次評定（4週ごと）
- ✓ 給与と経済（超簡易）
- ✓ 従僕システム（数値のみ、名前なし）
- ✓ エンディング（小頭到達で完結）

#### 実装しないもの
- ✗ 戦闘の詳細（自動判定のみ）
- ✗ 家臣団（従僕以外なし）
- ✗ 私事（村開発なし）
- ✗ 商人システム
- ✗ 婚姻システム
- ✗ 知行地の詳細

### プレイヤー初期状態

```
役職：徒士（指揮権なし）
家格：1（庶家下級）
知行地：なし
従僕：3名（名前なし、戦闘バフのみ）
給与：10貫/月
金：20貫
所属：加藤小頭の小隊（25人）の一員
```

### 出世時の変化

#### 徒士→馬上衆
```
給与：10貫 → 20貫/月
従僕：3名 → 2名
知行地：200石付与（月15貫）
馬の購入許可
```

#### 馬上衆→小頭
```
給与：20貫 → 40貫/月
従僕：2名 → 1名
知行地：200石 → 500石（月50貫）
指揮権：25人小隊を指揮
```

### 経済バランス

#### 徒士
```
収入：10貫/月
支出：
- 従僕3名：6貫（2貫/月/人）
- 自分：2貫
余剰：2貫/月

→ ギリギリ
→ 馬（30貫）買うには15ヶ月必要
```

#### 馬上衆
```
収入：20 + 15 = 35貫/月
支出：
- 従僕2名：4貫
- 自分：3貫
余剰：28貫/月

→ かなり余裕
→ 初めて投資可能
```

#### 小頭
```
収入：40 + 50 = 90貫/月
支出：
- 従僕1名：2貫
- 自分：5貫
余剰：83貫/月

→ 大きく余裕
→ 加速を実感
```

### ゲームループ

#### 1週間の流れ
```
1. ターン開始
   - 主君からの命令表示
   - ライバルの状況表示
   - 行動選択

2. 行動選択（公務）
   - 訓練 or 盗賊討伐 or 巡察
   
3. ターン実行
   - プレイヤーの行動結果
   - AIライバルの行動
   - 功績付与

4. ターン終了
   - 次週へ
```

#### 月次処理（4週ごと）
```
- 功績累計の確認
- 出世判定
- ライバルの出世判定
- 給与支給
- 評定（主君からのコメント）
```

#### ゴール
```
3年間（156週）で小頭到達
→ エンディング
→ プレイ時間：2〜3時間
```

## 5. 別紙A〜Mの扱い

### 別紙は「世界観の辞書」

別紙A〜Mには詳細な世界設定とシステムルールが記載されているが、
**Version 0.1ではほとんど使わない。**

### 参照する部分

#### 別紙C（役職システム）
- 役職名と階層のみ参照
- 徒士、馬上衆、小頭の定義

#### 別紙I（出世ロジック）
- 出世条件の考え方のみ参照
- 詳細な計算式は使わない

#### 別紙A（世界観方針）
- 「公務=功績」「私事=金」の原則
- 「家単位」の思想

### 使わない部分（Version 0.5以降）

- 別紙G・L（戦闘AI） → Version 0.5で実装
- 別紙H（経済・商人） → Version 1.0で実装
- 別紙K（婚姻） → Version 1.0で実装
- 別紙F（引き抜き） → Version 1.0で実装
- その他の詳細設定 → 段階的に実装

### 重要原則
**別紙に縛られない。Version 0.1の面白さを最優先。**

## 6. 重要な設計決定

### 従僕システム
- 給与：2貫/月（住み込みだから安い）
- 徒士：3名、馬上衆：2名、小頭：1名
- Version 0.1：名前なし、数値のみ
- Version 0.5：名前あり、個性あり、ドラマ性追加

### 時間単位
- 1ターン = 1週間
- 月次処理 = 4週ごと
- 年間 = 52週
- Version 0.1 = 3年間（156ターン）

### 知行地
- 徒士：なし
- 馬上衆：200石（月15貫）
- 小頭：500石（月50貫）
- Version 0.1：数値のみ、開発なし

### 戦闘
- Version 0.1：自動判定（成功/失敗のみ）
- Version 0.5：別紙G・Lに基づく詳細実装

## 9. このプロジェクトで避けるべきこと

### ❌ システムから考える
「戦闘システムはこうで...」
「経済システムはこうで...」
→ Version 0.9の失敗を繰り返す

### ❌ 細部に拘泥する
「従僕の給与は何貫？」
「知行地はいつもらう？」
→ 仮置きでいい、後で調整

### ❌ 完璧主義
「別紙通りに実装しないと...」
→ Version 0.1は最小限でいい

### ❌ 一気に作る
「全部実装してから確認」
→ 段階的に、テストプレイ必須

### ✅ やるべきこと
- プレイヤー体験から考える
- 最小ループを最優先
- 早くテストプレイ
- フィードバックで改善
- 段階的に拡張

# 📌 重要な注意事項

## 開発方針について

このサマリーに記載されたVersion 0.1〜2.0の開発方針は
**「現時点での仮案」**です。

### プロジェクト開始後の流れ

1. **GDD（ゲームデザインドキュメント）を作成**
   - ゲームの核心
   - 何が面白いか
   - プレイヤー体験

2. **GDDを元に開発方針を再検討**
   - Version 0.1のスコープは適切か
   - 期間は現実的か
   - 削るべき/追加すべき機能はないか

3. **相談しながら確定**
   - 「給与と知行地の関係は？」
   - 「徒士期間が暇すぎない？」
   - などの疑問を議論して解決

### サマリーの役割

- ✓ 失敗の教訓を伝える
- ✓ プロジェクトの方向性を示す
- ✓ 開発方針の「たたき台」を提供
- ✗ 確定版ではない

**確定版はプロジェクト内で作成します。**

### 特に要相談の項目

以下は仮案として記載していますが、プロジェクト内で改めて検討が必要です：

- Version 0.1の目標（徒士→小頭は適切？）
- 開発期間（3ヶ月は現実的？）
- 経済バランス（給与と知行地の関係）
- 従僕の扱い（数値のみ or 名前あり？）
- 初期状態（知行地なしでいい？）
- 公務の種類（3種類で足りる？）

これらの疑問を持ちながらプロジェクトを進めてください。
