# Version 0.17 要件定義書（改訂版）

## 1. 概要

### 1-1. バージョン目標

```
目標：v0.15 の「単調さ（クリック連打で最適化される状態）」を解消し、意思決定と学びが発生する状態にする
プレイ時間：徒士→小頭が30分前後
前提：Version 0.15
```

### 1-2. 方針（考える場所の設計）

```
- 「準備」だけでなく「戦闘中の操作（陣形/姿勢/撤退など）」でも考えさせる
- 金銭不足（装備/雇用/買収ができない）と、手柄（功績）を取りに行くリスク（損耗/負傷）のトレードオフを体感できる
- クリック回数を増やすのではなく、1回の意思決定の価値を上げる

- 下知（命令）は「討伐の指示」ではなく、上位の方針が段階的に具体化されて末端に伝達される
  - 第1段階：殿様→家老/部将（大評定）- 抽象的な方針（Why）
  - 第2段階：家老/部将→侍大将/足軽大将 - 方針の具体化（What）
  - 第3段階：侍大将/足軽大将→組頭/小頭 - 数値目標（How much）
  - 第4段階：組頭/小頭→馬上衆/徒士 - 実行手段（How）

- 方針は3種類（軍事寄り）
  - 治安回復（軍事・領内）
  - 軍備増強（軍事・準備）
  - 計略（軍事・対外）
  
- 役職による見え方の違い
  - 徒士/馬上衆：第4段階の命令のみ見える
  - 小頭：第3段階の命令を直接見える（目標数値が明示）
  - 組頭（将来）：第3段階 + 他小頭の進捗も見える
```

### 1-3. 解決したい問題（現状の課題）

```
- どの行動を選んでも結果が同じ（差分がない）
- 失敗や副作用がない（最適解が単一）
- 進捗が単調に足し算（考えなくても到達できる）
- ログが通知で終わり、次の意思決定に繋がらない
- 下知（命令）が固定化しており、同じ流れが周期的に繰り返される（通知→開始→討伐→通知...）

特に顕著：
- 盗賊討伐が「偵察なしで初手攻撃→押し切り」が成立する
- 戦闘が一律に見え、駆け引きや展開の幅が薄い
- 徒士→小頭までが短時間で到達する
```

### 1-4. 主要追加/変更（v0.17でやること）

| 項目 | 優先度 | 概要 |
|------|--------|------|
| 方針システムの実装 | 高 | 3つの方針（治安回復/軍備増強/計略）を実装し、月ごとに方針が変化する |
| 命令の4段階伝達 | 高 | 上位方針→目標→現場実行へ段階的に落ちる形へ変更する |
| 進捗の可視化 | 高 | 方針に対する進捗（ゲージ/数値）を可視化し、手段の成果を反映する |
| 複数の達成手段 | 高 | 同じ目標に対し、複数の手段があり、状況で最適解が変わる |
| 判断機会の設計 | 高 | 準備と実行のトレードオフ、複数の準備手段、準備なしも選択肢、状況の変化 |
| 目標達成の不確実性 | 高 | 安全策だけでは目標達成が確実ではない、運次第で未達もある |
| 報酬の段階性 | 高 | 平凡/優秀/卓越の3段階評価、予想外の成果には追加報酬 |
| 出世ペース調整 | 高 | リスク・リターンの設計、判断機会の密度向上で30分前後に調整 |
| 昇進システムの改善 | 高 | 昇進しても資源制約が続く、借金システムの活用、判断の質が変わる |
| ログの学び強化 | 中 | 「なぜそうなったか」「次は何を変えるべきか」が残るログにする |

## 2. 必須要件（確定事項）

### 2-1. 命令具体化の「作業化防止」要件

```
必須要件1：目標達成の不確実性
  - 安全策だけでは確実ではない
  - 最低限の行動では、運次第で未達もある

必須要件2：複数の達成手段がある
  - 目標に対して、異なる特性を持つ複数の手段が存在する
  - 状況に応じて最適な手段が変わる

必須要件3：報酬の段階性
  - 平凡/優秀/卓越で報酬が明確に異なる
  - 最低限で満足せず、上を目指す動機がある

必須要件4：予想外の展開
  - 予想を超える成果には追加報酬
  - 驚きと満足感が生まれる

必須要件5：能力と運のバランス
  - 短期的には運で逆転、長期的には能力が出る
  - 能力が低くても希望、能力が高くても油断できない

必須要件6：多様な評価軸
  - 行動種別で評価基準が異なる（能力依存/成果依存）
  - 多様な活躍の仕方がある
```

### 2-2. 「判断機会」の設計要件

```
必須要件1：準備と実行のトレードオフ
  - 準備に時間をかけると成功率は上がるが、機会を逃す可能性

必須要件2：複数の準備手段
  - 偵察以外にも準備の方法がある（買収/雇用/装備など）

必須要件3：準備なしも選択肢
  - 偵察なしで討伐することが合理的な状況がある

必須要件4：状況が変化する
  - 時間経過で状況が変わる（盗賊が移動/強化など）
```

### 2-3. 「命令のパターン化」防止要件

```
方針の種類：
1. 治安回復（軍事・領内）
   手段：巡察、情報収集、討伐など

2. 軍備増強（軍事・準備）
   手段：装備調達、兵の雇用、訓練、装備整備など

3. 計略（軍事・対外）
   手段：敵情視察、内通者への接触、情報収集、
         敵陣営への潜入、流言の流布、補給路の妨害、放火

命令伝達の4段階構造：
- 第1段階（殿様）：抽象的な方針（Why）
- 第2段階（家老）：方針の具体化（What）
- 第3段階（大将）：数値目標（How much）
- 第4段階（小頭）：実行手段（How）

方針の決定：
- 殿様が大評定で決定
- 毎月異なる方針が出る可能性
- プレイヤーには予測不可
```

### 2-4. 昇進の「作業効率化」防止要件

```
必須要件1（最重要）：昇進しても資源制約が続く
  - 配下を揃えないと主命達成が困難
  - しかし資金が不足
  - 出世するほど金の使い道が増える
  - 借金システムあり（既に実装済み）

必須要件2：できることが増えるが、失敗のコストも上がる
  - 新しい手段が使える
  - 失敗時の影響範囲が拡大

必須要件3：新しい種類の判断が必要になる
  - 徒士「何をするか」
  - 小頭「誰に何をさせるか」
  - 判断の質が変わる

プレイサイクル：
1. 主命を受ける
2. 配下/装備が足りない
3. 借金して調達
4. リスクを取って手柄を立てる
5. 報酬で返済
6. 繰り返すうちに出世
7. 気づいたら役職・権限が増えている
```

### 2-5. 出世ペースの要件

```
採用アプローチ：
- リスク・リターンの設計
  → リスクを取ると早く昇進、安全策だと遅い

- 判断機会の密度
  → 1回の意思決定の価値を上げ、判断の質で差が出る

目標プレイ時間：
徒士→小頭：30分前後
```

### 2-6. 昇進システムの基本要件

```
3要素の同時発生：
① できることが増える（手段の解禁）
② 見える情報が増える（上位の意図・状況）
③ 責任が重くなる（評価・未達の影響）

役職別の優先要素：
徒士→馬上衆：
  優先1：できること（戦闘での存在感）
  優先2：見える情報（上位方針が見え始める）
  優先3：責任（損耗が評価に響き始める）

馬上衆→小頭：
  優先1：できること（指揮権、戦闘中操作）
  優先2：責任（損耗効率が評価の中心）
  優先3：見える情報（目標数値が明確に）

小頭→組頭：
  優先1：責任（損耗効率、比較評価）
  優先2：できること（ローテ、規模拡大）
  優先3：見える情報（他小頭との比較）
```

### 2-7. 目標と評価の基本要件

```
二段階目標 + ライバル競争：
- 最低目標：運次第で未達もありうる、安全策だけでは確実ではない
- 挑戦目標：リスクを取れば達成可能、より大きな報酬
- ライバル競争：同期の小頭と比較される、勝っても報酬なし（誉め言葉のみ）

功績の3段階評価：
- 平凡：最低目標達成（基礎功績のみ）
- 優秀：挑戦目標達成（基礎功績 + ボーナス）
- 卓越：卓越目標達成（基礎功績 + 大ボーナス + 特別報酬（米など））

未達ペナルティ：
- 最低目標未達：平凡功績の4割減算
- 評価：「努力不足」
```

### 2-8. 成果計算の基本要件

```
基本式：
成果 = 能力 × ランダム係数

意図：
- 優秀な者は平均して高成果
- 運が悪い優秀 ≒ 運のいい無能（その時限定）
- 能力差がありつつ、運の要素もある

能力値範囲：
20〜100（確定済み）

討伐の特殊ルール：
- 討伐の評価は敵の規模に依存
- 能力は成功率に影響するが、成功後の評価は規模次第
- 無能でも大物を倒せば高評価

予想外のご褒美：
- 予想より大きい敵を倒した場合
- 追加報酬（米/資金/装備など）
```

## 3. スコープ（v0.17で実装する範囲）

### 3-1. 対象範囲

```

- 評価システム

  - 判断機会の密度向上
```

## 付録A：命令伝達の具体例（参考）

### 治安回復の場合

```
第1段階（殿様→家老）：
「近頃、領内で盗賊の噂が絶えぬ。
 百姓どもが安心して田畑に出られねば、年貢も上がらぬ。
 この夏までに、領内の治安を立て直すよう手配せよ。」

第2段階（家老→足軽大将）：
「殿様より治安回復の命あり。
 各大将は配下を指揮し、領内の盗賊を取り締まれ。
 今月末までに、領内巡察を強化し、盗賊の根拠を突き止め、
 討伐の準備を整えよ。」

第3段階（足軽大将→小頭）：
「殿様より治安回復の命があり、今月末までに成果を上げねばならぬ。
 治安の悪化を示す数値を、月末までに「60」まで回復させること。
 現在の治安数値は「40」である。あと「20」の改善が必要だ。
 手段は問わぬが、月末の報告で未達の場合は理由を述べよ。」

第4段階（小頭→徒士/馬上衆）：
「羽柴様より、今月は治安回復に努めよとの命があった。
 今週は、以下のいずれかを実施する：
 1. 巡察（領内を巡回し、盗賊の動きを探る）
 2. 情報収集（村人から聞き込み、盗賊の根拠を探る）
 3. 盗賊討伐（根拠が判明すれば、討伐に向かう）
 そなたは徒士/馬上衆として、いずれかに参加せよ。」
```

### 軍備増強の場合

```
第1段階（殿様→家老）：
「隣国との関係が険悪になっておる。
 いずれ戦となろう。今のうちに軍備を整えよ。
 兵を鍛え、武具を揃え、いつでも出陣できる体制を作れ。」

第2段階（家老→足軽大将）：
「殿様より軍備増強の命あり。
 各大将は配下の装備を点検し、不足があれば補充せよ。
 また、兵の訓練を強化し、戦闘力を高めよ。」

第3段階（足軽大将→小頭）：
「殿様より軍備増強の命があり、今月末までに戦闘準備を整えよ。
 軍備充実度を、月末までに「60」まで高めること。現在は「40」である。
 装備購入には費用がかかる。無駄遣いは避けよ。」

第4段階（小頭→徒士/馬上衆）：
「羽柴様より、今月は軍備増強に努めよとの命があった。
 今週は、以下のいずれかを実施する：
 1. 装備調達（武器商人から装備を購入）
 2. 訓練（小隊での戦闘訓練を実施）
 3. 兵の募集（新しい徒士を雇用）
 4. 装備整備（既存装備の修理・手入れ）
 そなたは徒士/馬上衆として、いずれかに参加せよ。」
```
